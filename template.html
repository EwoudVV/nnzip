  How does this work?
  
Both you and the sender's machine query GPT-3.5 for its
  top-5 most likely next tokens at each position. The compressed payload only
  records which rank (0-4 or "not in top-5") the real next token had -- a
  handful of bits per token. When the real token wasn't in the top-5, we also
  embed it as a literal string. Reconstructing the file requires the same
  model and an API key; the content itself is not encrypted.
